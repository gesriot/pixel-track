# Pixel Track

Приложение на Python/PySide6 для ручного покадрового измерения движения объектов по серии изображений.

Текущий срез проекта после Спринта 6:
- открытие папки с кадрами;
- поддержка стандартных форматов изображений Qt;
- натуральная сортировка последовательности кадров;
- навигация по кадрам;
- масштабирование с сохранением зума между кадрами;
- настройка FPS из интерфейса;
- режим калибровки на текущем кадре;
- ввод длины калибровочного отрезка в метрах;
- отображение пиксельной длины и масштаба в правой панели;
- ручная разметка `Previous` и `Current` точки движения;
- расчёт расстояния шага и скорости в `м/с` для одного объекта;
- журнал измерительных шагов;
- график изменения скорости во времени;
- сохранение и загрузка проекта в `JSON`;
- экспорт измерений в `CSV`;
- переносимые project-файлы, которые можно двигать вместе с папкой кадров;
- горячие клавиши для навигации и смены режимов;
- режим `Edit Handles` для перетаскивания калибровки и измерительных точек прямо на кадре;
- небольшой кеш соседних кадров для более быстрой перелистки длинных серий.

## Запуск

Если окружение уже создано через `uv`, достаточно:

```bash
source .venv/bin/activate
python main.py
```

После синхронизации зависимостей через `uv sync` можно запускать и так:

```bash
uv run pixel-track
```

## Sprint 6

Через меню `File` по-прежнему доступны:
- `Open Project...`
- `Save Project`
- `Save Project As...`
- `Export Measurements CSV...`

В интерфейсе дополнительно появились:
- клавиши `Left/Right` для соседних кадров;
- `PageUp/PageDown` для прыжка на 10 кадров;
- `Home/End` для первого и последнего кадра;
- `N`, `C`, `P`, `M`, `E` для режимов `Navigate`, `Calibrate`, `Mark Previous`, `Mark Current`, `Edit Handles`;
- `Ctrl+=`, `Ctrl+-`, `Ctrl+0` для управления масштабом.

Проект по-прежнему сохраняет:
- список кадров;
- папку с исходными изображениями;
- FPS;
- калибровки по кадрам;
- точки `Previous` и `Current`;
- историю, из которой заново строятся таблица и график.
